# Pokemon Quiz — идеи для следующих режимов

Бэклог категорий, которые можно добавить после типового квиза. Каждая — новая ось
(не просто «ещё один пул»): другой канал восприятия или другой тип знания.

---

## 1. «Силуэт» — Who's That Pokémon?
**Механика:** картинка 100% залита чёрным (`filter: brightness(0)`), 4 варианта
имени. При ответе — драматичное «проявление» силуэта (тот самый момент из аниме).

**Почему кайфово:** мгновенная ностальгия, 8-летка узнаёт формат с первой секунды.
Картинка и так в приложении → реализация = 10 строк CSS + transition на reveal.

**Ось:** форма без цвета/деталей.
**Сложность реализации:** тривиально.

---

## 2. «Крик» — покемон по звуку
**Механика:** кнопка 🔊 (без картинки) → проигрывается крик → 4 варианта имени.
PokéAPI отдаёт `.ogg` напрямую:
`https://raw.githubusercontent.com/PokeAPI/cries/main/cries/pokemon/latest/{id}.ogg`

**Почему кайфово:** совершенно другой сенсорный канал, дети обожают смешные звуки
покемонов (Джигглипафф, Псайдак, Вульпикс — узнаваемы с первой секунды). Можно в
конце подложить **свой кастомный звук** — как Кира/Онекс/Мисти говорят имя
покемона. Точно попадёт в восторг.

**Ось:** аудио.
**Сложность реализации:** низкая (предзагрузить ogg, Web Audio).
**Риск:** часть криков похожи между собой → вручную whitelist-нуть 40 самых
характерных.

---

## 3. «Наши покемоны» — из Pokemon Stories
**Механика:** пул = только покемоны, которые реально появились в Ep1-8 (Пикачу,
Чармандер, Сквиртл, Иви, Сикстейлс, Пиджеотто, Бульбазавр, Абра, Баттерфри,
Псайдак, Спироу, Мэнки, Арбок, Лапрас, Катерпи, Генгар, Арканайн, Кроубат — ~18).
Часть вопросов — **сюжетные**: «Какой покемон дал Ромусу подсказку с двухсекундным
окном?» (Бульбазавр, Ep4). Можно показывать **иллюстрацию из книжки** и
спрашивать «какой покемон на картинке?».

**Почему кайфово:** максимально персональная категория, уникальна для этого
продукта. Дети флексят знанием **собственных историй** — это совсем другое
ощущение, чем угадать Мьюту. Автоматически связывает секции сайта «эпизоды» и
«квиз».

**Ось:** нарратив + память.
**Сложность реализации:** средняя — надо написать 20–30 сюжетных вопросов
вручную. Но это разовая работа и она обогащает весь проект (привязки к
эпизодам = контент-граф на будущее).

---

## 4. «Ультра-зум» — по детали
**Механика:** случайный кроп 10–15% артворка масштабируется ×4–8 → угадай по
глазу, лапе, узору хвоста. 4 варианта имени.

**Почему кайфово:** режим «детектива», детям 7–8 это ужасно интересно
(как "I spy"). Бесконечно реиграбелен: один и тот же покемон даёт новый вопрос
при новом кропе. Лёгкие кропы (мордашка Пикачу) = подсказка для маленьких,
сложные (кусок живота Мьюту) = вызов.

**Ось:** визуальный детализм / мелкая моторика взгляда.
**Сложность реализации:** низкая (CSS `background-position` + `background-size`
на том же URL). Нужен небольшой whitelist «интересных» координат на покемона,
иначе попадаются пустые зелёные пиксели.

---

## 5. «Эволюции» — цепочки
**Механика:** «Во что превращается Чармандер?» / «Кто эволюция Баттерфри?» /
«Какая средняя форма между Бульбазавром и Венузавром?». 4 картинки-варианта.

**Почему кайфово:** учит **core-концепт** покемонов, который дети каждый день
видят в книжке (Чармелеон в Ep19, Вартортл в Ep20 — тема сезона!). После
прохождения квиза ребёнок реально начинает разбираться в эволюциях, а не просто
узнавать картинки.

**Ось:** логика отношений / память цепочек.
**Сложность реализации:** средняя (PokéAPI отдаёт evolution-chain, но нужны
fallback-ветки для Иви, разделить «первая-средняя-финальная»).
